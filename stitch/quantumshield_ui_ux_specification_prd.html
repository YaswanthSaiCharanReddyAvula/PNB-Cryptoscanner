# QuantumShield: Quantum-Safe Cryptography Migration Platform - UI/UX Specification ## 1. Information Architecture & Route Map A comprehensive platform management structure. ### Routes - `/` - Overview (Executive Dashboard) - `/inventory/runs` - Inventory Runs (History) - `/inventory/assets` - Assets Inventory - `/inventory/findings` - Crypto Findings (Normalized) - `/inventory/cbom` - Crypto Bill of Materials (CBOM) - `/quantum/overview` - Quantum Risk Overview - `/quantum/threat-map` - Threat Map - `/quantum/hndl` - HNDL Exposure - `/migration/planner` - Migration Planner (Phases 0-3) - `/migration/tasks` - All Migration Tasks - `/migration/waivers` - Waivers / Exceptions Management - `/policy` - Org Crypto Policy & Standards - `/reporting/executive` - Executive Report View - `/reporting/exports` - Export Center - `/admin/integrations` - Admin Settings (Users, GitHub/Jira) --- ## 2. Global Layout Spec - **Sidebar Nav:** Fixed, collapsible. Categorized by functional areas (Inventory, Quantum Risk, Migration, etc.). - **Top Bar:** - **Global Search:** Command+K style. - **Notifications:** Bell icon with badge for system events. - **User Profile:** Displays role (Admin/Employee). - **Breadcrumbs:** Auto-generated based on route. - **Main Content:** Scrollable area with consistent 24px-32px padding. --- ## 3. Visual Language & Tokens ### Severity Taxonomy - **CRITICAL:** `#E11D48` (Rose-600) - **HIGH:** `#EA580C` (Orange-600) - **MEDIUM:** `#D97706` (Amber-600) - **LOW:** `#2563EB` (Blue-600) - **SAFE:** `#059669` (Emerald-600) ### Quantum Status - **VULNERABLE:** High-intensity red/pink. - **PARTIALLY_SAFE:** Amber. - **QUANTUM_SAFE:** Solid Emerald/Green. --- ## 4. Reusable Components Catalog - `PageHeader(title, subtitle, actions)` - `KpiCard(label, value, delta?, severity?)` - `RiskBadge(level: 'CRITICAL' | 'HIGH' | 'MEDIUM' | 'LOW' | 'SAFE')` - `QuantumStatusBadge(status: 'VULNERABLE' | 'PARTIALLY_SAFE' | 'QUANTUM_SAFE')` - `DataTable(columns, data, filters, rowActions, bulkActions)` - `EvidencePanel(source, items)` - For displaying line numbers or TLS details in a code-snippet style drawer. - `DetailDrawer(title, tabs, footerActions)` - Side panel for deep dives. --- ## 5. API Data Expectations (High-Level) - **Asset:** `id, name, type, environment, criticality, quantum_status, highest_risk, last_assessed` - **Finding:** `id, title, asset_id, category, algorithm, impact (Shor/Grover/HNDL), risk_level, confidence, evidence` - **MigrationTask:** `id, phase, status, title, asset_id, due_date, owner_id, dependencies` - **Waiver:** `id, requestor, reason, expiry, impacted_assets[], status` --- ## 6. MVP Implementation Order 1. **Foundation:** Layout + Sidebar + TopBar. 2. **Phase 1: Inventory:** Runs history -> Assets -> Findings (The data source). 3. **Phase 2: Risk Assessment:** Executive Dashboard -> Threat Map -> HNDL. 4. **Phase 3: Migration:** Planner -> Tasks -> Waivers (The action layer). 5. **Phase 4: Admin & Reporting:** Policy -> Exports -> Integrations.
